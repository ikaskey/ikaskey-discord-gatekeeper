# 段階移行ガイド（既存サーバーへの導入）

既にメンバーがいる Discord サーバーへ、いきなり大量キックを起こさず段階的に導入するための手順。

## 大前提（安全性）

- **定期検証スイープ（退会連動）は `Link`（＝認証済み）レコードのある人しか検査しません。**
  既存の未認証メンバーは `Link` が無いため、デプロイしても検査もキックもされません。
- リスクのある操作（チャンネルゲート・未認証キック）は、こちらが明示的に進めた段階でのみ発生します。

## Phase 0 — 並走デプロイ（誰も影響を受けない）

1. `.env` を用意（Discord 認証情報 ＋ `ADMIN_COOKIE_SECRET`）。**念のため `SWEEP_ENABLED=false`** にしてキックを完全停止。
2. `docker compose up -d --build`、cloudflared で `PUBLIC_BASE_URL` を web に向ける。
3. `pnpm bot:deploy-commands`（`/verify-panel` `/rolemap` `/migration-status` を登録）。
4. 認証チャンネルで `/verify-panel` を実行してパネル設置。
5. この時点ではチャンネル権限は変更せず、全員これまで通り利用可能。

> `SWEEP_ENABLED=false` の間は、認証済みユーザーの退会連動キックも止まります。Phase 2 以降で `true` に戻します。

## Phase 1 — 認証の周知と猶予期間

- 既存メンバーに認証パネルから認証してもらう（→ `Link` 作成＋会員ロール付与）。
- **進捗確認**: `/migration-status` で「認証済み / 未認証 / 除外 / Link件数」と未認証者リストを確認。
- 猶予期間（例: 2〜4週間）を設定し、アナウンス・リマインド。
- 退会連動を再開してよくなったら `SWEEP_ENABLED=true` に戻す（認証済みユーザーのみ対象なので既存未認証者には無影響）。

## Phase 2 — チャンネルのソフトゲート（M2）

会員チャンネルを**会員ロール必須**にする。未認証者はキックされず、認証すれば即復帰できる非破壊な方式。

### 推奨手順（権限ベース・追加ロール不要）

1. **`@everyone` ロール**: 会員向け各チャンネル（カテゴリ）で `チャンネルを見る (View Channel)` を **拒否**。
2. **会員ロール（`VERIFIED_ROLE_ID`）**: 同チャンネル（カテゴリ）で `チャンネルを見る` を **許可**。
   - カテゴリに設定 → 配下チャンネルへ継承させると楽。
3. **認証チャンネル**: `@everyone` に `チャンネルを見る` を **許可**（未認証者がここだけ見えるように）。会員ロールにも許可。
4. 結果:
   - 未認証メンバー → `#認証` だけ見える。認証すると会員ロールが付き、会員チャンネルが開く。
   - 新規参加者 → 自動的に未認証状態（会員ロール無し）= `#認証` のみ。
5. 確認: テスト用アカウントや権限プレビューで「未認証から見える範囲」をチェック。

> ロールの並び順は **Bot ロールを会員ロールより上**に（付与・剥奪のため）。

## Phase 3 — 未認証メンバーのキック（期限後・任意）

猶予期間が終わったら、未認証メンバー（`Link` 無し・Allowlist 外）をキックする運用に移行できます。

- まず `/migration-status` で対象規模を把握。
- 残しておきたい個別メンバーは管理画面の **除外リスト（Allowlist）** に追加。
- ※ 未認証キックの**自動機構（dry-run 付き）は Phase 3 到達時に追加実装**します（現状はゲート＋手動運用）。

## 関連する環境変数

| 変数                      | 役割                                             |
| ------------------------- | ------------------------------------------------ |
| `SWEEP_ENABLED`           | 退会連動キックのキルスイッチ（移行中は `false`） |
| `SWEEP_FAILURE_THRESHOLD` | 慎重にするなら `2` 以上（連続失敗で確定キック）  |
| `VERIFIED_ROLE_ID`        | 会員ロール。チャンネルゲートの可視制御に使う     |

## 便利コマンド

- `/migration-status` — 認証済み/未認証の集計と未認証者リスト（管理者のみ・読み取り専用）
- `/rolemap` / 管理画面（`/admin`） — ロール連動・除外リスト・監査ログ
